### 📌 Anthropic 公开 Claude Code 退化事后复盘：三处改动 + 全员 usage 重置
**Anthropic** 于 **4 月 23 日**发布官方工程博客 **"An update on recent Claude Code quality reports"**，首次**完整承认**过去一月 Claude Code / Claude Agent SDK / Claude Cowork 的质量下滑源自**三处独立改动**：
- **3 月 4 日**：Claude Code 默认 **reasoning effort 由 high 降为 medium**（为缓解部分用户"长时间高 effort"体验），**4 月 7 日**回滚；
- **3 月 26 日**：**超过 1 小时空闲**会话清理旧 thinking 的补丁存在 **caching bug**，实际变成"**每一轮都清一次**"，导致**遗忘且重复**，**4 月 10 日**修复；
- **4 月 16 日**：为降低冗余添加的 **verbosity 系统提示**与其他 prompt 改动叠加**损害了 coding 质量**，**4 月 20 日（v2.1.116）**回滚。

同日 Anthropic 宣布**对所有订阅者重置当前周期的 usage 限额**作为补偿，并公开 **"我们会在出现 silent quality regression 时向用户更透明地通报"**的内部承诺。The Register / VentureBeat 将其评为"**LLM 产品史上第一份公开的质量事后复盘**"，**定调 2026 年大模型厂"可靠性 + 可解释性"第二战场**。（来源：[Anthropic Engineering Blog](https://www.anthropic.com/engineering/april-23-postmortem) · [VentureBeat](https://venturebeat.com/technology/mystery-solved-anthropic-reveals-changes-to-claudes-harnesses-and-operating-instructions-likely-caused-degradation) · [The Register](https://www.theregister.com/2026/04/23/anthropic_says_it_has_fixed/)）
