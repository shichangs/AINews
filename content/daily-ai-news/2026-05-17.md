## 2. 主要 AI Labs 动态：安全事件

### 🔥 OpenAI 5/15 披露 TanStack npm 供应链攻击，4 款 macOS 应用签名证书全部轮换

5/15 周五 11:08 UTC OpenAI 在官方安全博客披露：**2 名员工**设备在 5/11 的 **TanStack npm 供应链攻击**中被 "Mini Shai-Hulud" 蠕虫感染。攻击者对这两台设备可触达的有限内部代码仓库执行"以凭证为目标的外泄活动"，OpenAI 表示**仅有限的凭证物料**被成功外泄，没有客户数据、生产系统或对外软件受影响。事件触发了 ChatGPT Desktop、Codex App、Codex CLI、Atlas 这 **4 款 macOS 应用**签名证书的全部轮换，OpenAI 给用户 **6/12** 之前的更新窗口。

The Register 与 The Hacker News 5/15 复盘指出：OpenAI 受影响的两台设备"尚未完成"3 月底 Axios 供应链事件后引入的新型软件包管理保护的灰度部署，正是这道时间缝把恶意依赖放进了内网。Socket 同周确认 TanStack 此次有 **84 个**恶意包版本横跨 **42 个** `@tanstack/*` 包名被发布，专门窃取 GitHub token、云密钥、npm 凭证与 CI/CD 认证物料。该次 OpenAI 暴露面与 5/12 SAP-related npm 包遭袭事件被研究员归为同一 TeamPCP 组织持续行动。

（来源：[OpenAI 官方披露](https://openai.com/index/our-response-to-the-tanstack-npm-supply-chain-attack/) · [The Register](https://www.theregister.com/security/2026/05/15/openai-caught-in-tanstack-npm-supply-chain-chaos-after-employee-devices-compromised/5241019) · [The Hacker News](https://thehackernews.com/2026/05/tanstack-supply-chain-attack-hits-two.html)）

## 3. AI Agent 与企业部署

### 🔥 GOV.UK Chat 5/15 全量上线英国政府 App，由 Claude 驱动

5/15 周五英国 Government Digital Service（GDS）宣布 **GOV.UK Chat** 在 GOV.UK App 内向全量用户开放，结束 3/26 以来的软启动阶段。底层 LLM 为 **Anthropic Claude**，开发周期约两年半，知识覆盖 gov.uk 网站全部约 **80,000 页**内容。GDS 把它定位为"目前最大规模的政府自建聊天工具"，可在 24×7 期间用自然语言为公民解答个人税务、驾驶 / 交通、福利三类咨询——这三类在软启动期占查询量绝大多数。

发布同日 LBC 报道指 GOV.UK Chat 在测试中给出过"误导性"的税务答复，尽管 GDS 自报答案准确率约 **90%**。The Register 同日跟踪指出 GOV.UK Chat 在硬启动前的两年半测试覆盖 200,000+ 用户，其中 60% 把答案评为"满意或非常满意"。这是 Anthropic Claude 落地海外政府公民服务一线场景的首例公开部署，与 Anthropic 5/14 与 PwC、Gates Foundation 双线企业 / 慈善扩张构成节奏对应。

（来源：[GDS 官方公告](https://gds.blog.gov.uk/2026/05/14/gov-uk-chat-launches/) · [PublicTechnology 跟踪](https://www.publictechnology.net/2026/05/15/society-and-welfare/government-ai-chatbot-goes-live-across-gov-uk-app/) · [The Register](https://www.theregister.com/public-sector/2026/05/15/britains-latest-civil-servant-is-a-chatbot-trained-on-govuk-misery/5240955)）
