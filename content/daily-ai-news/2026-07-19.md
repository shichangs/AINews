### 🔥 Anthropic 与 Meta 洽谈约 $100 亿算力租赁，Meta 云业务浮出水面

Meta 若达成这笔约 **$100 亿**的算力租赁，将拿下其云算力业务的首个旗舰客户。据 CNBC 7/17 独家报道、CNN 随后跟进确认：Anthropic 于 6 月提出该方案，基本结构为两年期按月付费、任一方可提前退出；谈判处于极早期、可能最终不成，双方均拒绝置评。背景：数周前 Anthropic 刚与 SpaceX 达成使用 Colossus 1 数据中心算力的类似协议；Zuckerberg 5 月曾表示考虑对外出售算力，以证明 AI 资本开支能产生广告之外的收入。竞争关系上，Meta 自研 Llama 与 Claude 直接竞对，若成交将同时充当 Anthropic 的基础设施供应商——算力短缺正让此类交叉安排常态化。下一节点：谈判无公开时间表，可观察 7 月底 Meta 财报季管理层是否回应云业务问题。

（来源：[CNBC](https://www.cnbc.com/2026/07/17/anthropic-meta-ai-compute.html) · [CNN Business](https://www.cnn.com/2026/07/17/tech/meta-anthropic-ai-cloud-computing)）

### Microsoft 开发 Project Perception：多模型路由压成本，对标 Mythos

Project Perception 拟 7 月内上线，将竞对模型捆进 Microsoft 安全产品线。据 The Information 7/17 独家报道：该工具部署在企业 IT 系统内扫描、定位并修复漏洞，按任务把查询路由给 Microsoft、OpenAI、Anthropic 三家模型以压低成本——TechRepublic 引述估算称 Anthropic Mythos 的 API 成本较 Opus 高 **100%**、较 OpenAI GPT 系列高 82%（估算口径，未注明具体型号）。背景：Nadella 7/12 在 X 发文警告部分 AI 厂商在"用客户数据改进自身模型"（TechCrunch 7/13 报道转述）；Microsoft 面临的出口审查压力小于 Anthropic，欧洲或成切入口。下一节点：7 月内的正式发布与定价披露。

（来源：[The Information](https://www.theinformation.com/briefings/exclusive-microsoft-preps-mythos-like-ai-bug-finder) · [TechRepublic](https://www.techrepublic.com/article/news-microsoft-project-perception-ai-security-tool/)）

## 2. 中国 AI 产业

### 📌 WAIC 展商结构：基础大模型仅 18 家，"摸高"集中到头部

WAIC 大模型赛道 **130 家**展商中基础大模型仅 **18 家**，63.8% 转向 Agent 标签。甲子光年对全部 1,000 余家参展主体的统计显示，83 家以 AI Agent 或智能应用为主标签，"造模型"到"用模型"的迁移已成结构性事实；与此同时头部厂商继续向参数"摸高"：Kimi K3 的 2.8T 参数发布后（详见 07-17 归档），传 MiniMax 下一代模型代号 **M3 Pro**、总参数超 2 万亿并将开源，侧重复杂推理与多步骤 Agent 任务——MiniMax 未确认该信息；科大讯飞称已有万亿参数模型布局。第三方视角：Atreides Management CIO Gavin Baker 评 K3 称，其可能阻止 2–3 家"拥有 90%+ 推理利润率"的前沿实验室垄断生态（条件式表述）。下一节点：7/20 闭幕前，MiniMax 多模态模型 H3 与商汤 SenseNova U1 Pro 的现场发布细节。

### Claude Code v2.1.212：/fork 后台会话、WebSearch 每会话上限

Claude Code v2.1.212 给 WebSearch 设每会话默认 **200 次**上限，高频搜索的自动化流程需自查。7/17 发布（据 GitHub release tag，列表页缓存滞后）：/fork 将当前对话复制为独立后台会话（在 claude agents 中单独一行），原会话内子代理改名 /subtask；新增 claude auto-mode reset 一键恢复默认 auto-mode 配置；WebSearch 上限用于阻断失控搜索循环，可经 CLAUDE_CODE_MAX_WEB_SEARCHES_PER_SESSION 调整；另修复 Chrome 扩展启用但浏览器未运行时的启动挂起。行动项：依赖高频 WebSearch 的调度任务与脚本，升级前先确认 200 次默认上限是否够用。
