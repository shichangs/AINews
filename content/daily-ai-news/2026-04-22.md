## 1. 主要 AI Labs 动态

### 🔥 SpaceX 与 Cursor：$10 亿合作框架与 $60 亿收购选择权，IPO 之后行权
**SpaceX** 于 4 月 21 日深夜披露与 AI 编程初创 **Cursor**（母公司 Anysphere）的合作框架，4 月 22 日经 TechCrunch、Fortune、Bloomberg 等媒体跟进确认完整结构：一笔 **$100 亿** 即时合作协议——Cursor 的产品与开发者分发能力对接 SpaceX **Colossus** 超级计算机（SpaceX 称其等效约 100 万张 Nvidia H100），重点开发"编码与知识工作 AI"。与此并行的是一份最高 **$600 亿** 的收购选择权，行权窗口延至 SpaceX 今年夏季 IPO 之后——延期主因是 SpaceX 不想在上市前修改保密财务披露，并希望用上市后股票来支付对价。

该框架直接打断 Cursor 原本同周即将关闭的 $20 亿融资、$500 亿估值的新轮，TechCrunch 标题指 SpaceX "用 $600 亿 buyout 把 $20 亿 fundraise 撬翻"。结合 Mistral 联合创始人 Devendra Chaplot 三月加盟 xAI、以及 xAI 已并入 SpaceX，Musk 阵营在前沿模型、编码 Agent、超算三条线再次以资本动作锁定关键资产；外界关注：Cursor 一旦被 SpaceX 收购，Anthropic Claude 在 Cursor 内的默认模型地位是否还能维持。（来源：[Bloomberg](https://www.bloomberg.com/news/articles/2026-04-21/spacex-says-has-agreement-to-acquire-cursor-for-60-billion) · [CNBC](https://www.cnbc.com/2026/04/21/spacex-says-it-can-buy-cursor-later-this-year-for-60-billion-or-pay-10-billion-for-our-work-together.html) · [TechCrunch](https://techcrunch.com/2026/04/22/how-spacex-preempted-a-2b-fundraise-with-a-60b-buyout-offer/)）

### Anthropic Claude Code Pro 订阅档进行 2% 测试，发现后即时回滚
**Anthropic** 于 4 月 22 日短暂调整 `claude.com/pricing` 页面——新注册 Pro（**$20/月**）用户中约 **2%** 样本被取消"默认包含 Claude Code"；现有 Pro 与 Max 订阅者不受影响。改动数小时内被开发者社区（Hacker News、Reddit、Simon Willison 博客、The Register）发现并集中讨论，Anthropic Head of Growth Amol Avasare 当日在 X 公开承认这是"面向定价的小范围试探"，未发改版日志亦无正式公告；同日页面回滚，Pro 列重新勾选 Claude Code。背景：Claude Opus 4.7 上线后重度 Claude Code 用户的人均用量大幅上行，挤压 Pro 档毛利。该试探显示 Anthropic 正在评估"Claude Code 向 Max 档（$100 / $200）集中"的路线，但当前不打算明面切换。

## 2. AI Agent

### 🔥 Salesforce 与 Google Cloud：Agentforce 原生跑 Gemini、Lakehouse Zero-Copy 直读
**Salesforce** 与 **Google Cloud** 于 4 月 22 日联合宣布扩大合作，三项落地能力分阶段释出：

- Agentforce 与 **Atlas Reasoning Engine** 原生支持 Gemini 模型——Agent 可在文本、图像、视频模态间统一推理并调用多年客户历史；交付窗口 2026 年 5 月
- Zero-Copy 与 Google Lakehouse：Agentforce 可直接读写 Lakehouse 中的客户数据，无需复制、无需出域，降低数据迁移与合规风险；交付窗口 2026 年下半年
- Slack 与 Gemini Enterprise / Google Workspace 互通：Agentforce 与 Gemini Enterprise Agent 可在 Slack 与 Workspace 中协同跨平台动作；Slack 内的 Gemini Enterprise 4 月已可用

这是继 Anthropic Claude 已嵌入 Salesforce Agentforce、以及 Microsoft 365 Copilot 跨产品集成之后，**前沿 LLM 双方第一次在企业 Agent 层级正式互联**——双方都把对方已有的企业数据基座视为比"绑死自家堆栈"更重要的目标用户面。配套的还有 Google Cloud Partner of the Year 2026 颁给 Salesforce 与 Cognizant 等合作伙伴。（来源：[Salesforce 新闻稿](https://www.salesforce.com/news/press-releases/2026/04/22/salesforce-google-cloud-launch-new-integrations-deep-context/) · [Google Cloud Press Corner](https://www.googlecloudpresscorner.com/2026-04-22-Salesforce-and-Google-Cloud-Enable-AI-Agents-to-Act-Across-Both-Platforms-with-Deep-Context-and-End-to-End-Workflows) · [SiliconANGLE](https://siliconangle.com/2026/04/22/salesforce-google-partner-agentic-cross-platform-collaboration/)）

### Google Cloud Wiz Agentic Defense：三大新安全 Agent 与跨云 AI-APP
**Google Cloud** 于 4 月 22 日发布 **Agentic Defense** 体系，把 Google Threat Intelligence、Security Operations 与 Wiz 云安全平台合为单一架构，并新增三个 SOC 端 Agent：

- Threat Hunting Agent：自动猎捕未知攻击图谱、绕过现有规则的隐蔽行为
- Detection Engineering Agent：自动识别检测覆盖缺口、生成针对性检测逻辑
- Third-Party Context Agent：将外部威胁情报与企业资产关联，输出可执行风险上下文

存量 Triage 与 Investigation Agent 已累计处理 **500 万** 条告警，把典型人工分析的 **30 分钟** 压缩到 **60 秒**；Wiz AI-Application Protection Platform（AI-APP） 同期覆盖到 AWS AgentCore、Azure Copilot Studio、Salesforce Agentforce、Databricks——首次以"AI 应用与 Agent 平台"为防护单元。Google Cloud 在博客中将该方向定调为从人主导防御到人在环、再到 AI 主导加人审核的三段式演进。（来源：[Google Cloud Blog](https://cloud.google.com/blog/products/identity-security/next26-redefining-security-for-the-ai-era-with-google-cloud-and-wiz) · [SiliconANGLE](https://siliconangle.com/2026/04/22/google-cloud-next-new-security-operations-agents-wiz-integrations-agent-governance-tools/) · [The Register](https://www.theregister.com/2026/04/22/google_unleashes_even_more_ai/)）

### ServiceNow 与 Accenture：联合 Forward Deployed Engineering 团队下场
**ServiceNow** 与 **Accenture** 于 4 月 22 日宣布共建 Forward Deployed Engineering（FDE）团队，把 ServiceNow AI-native FDE 工程师与 Accenture 行业 FDE 顾问联合派驻进入大客户环境，直接在 ServiceNow AI Platform 上为客户搭建生产级 Agent 工作流；首批覆盖金融服务、医疗、电信、公共部门四个垂直行业。这一动作与 Google Cloud 同日宣布的 **$7.5 亿** Agent 合作伙伴基金、以及与 Accenture、Capgemini、Cognizant、Deloitte、Devoteam、HCLTech、TCS 七家 GSI 的 FDE 联营计划同步——**"派工程师驻场而非交平台"** 正成为 Agent 落地的主流模式，对纯软件许可式销售形成结构性挤压。（来源：[ServiceNow Newsroom](https://newsroom.servicenow.com/press-releases/details/2026/ServiceNow-and-Accenture-launch-forward-deployed-engineering-program-to-scale-agentic-AI-across-the-enterprise/default.aspx) · [Accenture Newsroom](https://newsroom.accenture.com/news/2026/servicenow-and-accenture-launch-forward-deployed-engineering-program-to-scale-agentic-ai-across-the-enterprise) · [Pulse2](https://pulse2.com/servicenow-and-accenture-forward-deployed-engineering-program-launched-to-scale-agentic-ai/)）

### Cognizant Agentic Retail CX：基于 Gemini Enterprise 的零售客服 Agent
**Cognizant** 于 4 月 22 日联合 Google Cloud 推出 **Agentic Retail CX**——一套基于 Gemini Enterprise 的零售客服 Agent 解决方案，主打 **70–85%** 自助化拦截率，含 omnichannel 主动外联（弃购车提醒、促销触达、反馈采集）与预训练零售数据模型（号称把自定义开发数月压缩到上线即可）。Cognizant 与 Google Cloud 同步官宣联合成立 Gemini Enterprise Practice 专项，并获 Google Cloud Partner of the Year 2026（数据分析、医疗与生命科学两个类别）。这是继 Merck 与 Google Cloud 多年协议之后，Cloud Next '26 上落地的另一笔头部 GSI 与行业垂直 Agent 合同。（来源：[Cognizant 新闻稿](https://news.cognizant.com/2026-04-22-Cognizant-and-Google-Cloud-Bring-Agentic-AI-to-Retails-Most-Critical-Customer-Moments-with-Gemini-Enterprise) · [PR Newswire](https://www.prnewswire.com/news-releases/cognizant-and-google-cloud-bring-agentic-ai-to-retails-most-critical-customer-moments-with-gemini-enterprise-302749783.html)）

## 3. AI 产品更新

### Qwen3.6-27B 开源发布：27B Dense 模型在 SWE-bench 上 77.2 分
**阿里通义** 于 4 月 22 日在 Hugging Face、ModelScope 与 GitHub（Apache 2.0 协议）上线 **Qwen3.6-27B**——一颗 27B 稠密模型，主打"旗舰级 Agentic 编码"。关键能力：

- 架构：64 层 hybrid Gated DeltaNet + Gated Attention——每 4 层中 3 层为线性注意力，搭配 Multi-Token Prediction（MTP）加速推理
- 上下文：原生 **262,144** tokens，可外推至约 **101 万** tokens
- 基准：**SWE-bench Verified 77.2**、**Terminal-Bench 2.0 59.3**（与 Claude 4.5 Opus 同档）、QwenWebBench 1,487——在多项 Agentic 编码、工具调用任务上超过自家 397B-A17B MoE 旗舰

伴随发布的还有 Qwen3.6-35B-A3B（4 月 16 日已上线）与一系列 Coder、Image、VL 子模型。Simon Willison 博客与 MarkTechPost 把这次发布概括为**用约 1/15 总参数量打到同档 MoE 模型水位**，并指出阿里同期在自家旗舰 Qwen3.6-VL Pro 上转向闭源权重——开源、闭源双轨走线更明确。（来源：[Qwen Blog](https://qwen.ai/blog?id=qwen3.6-27b) · [Hugging Face 模型卡](https://huggingface.co/Qwen/Qwen3.6-27B) · [MarkTechPost](https://www.marktechpost.com/2026/04/22/alibaba-qwen-team-releases-qwen3-6-27b-a-dense-open-weight-model-outperforming-397b-moe-on-agentic-coding-benchmarks/)）
