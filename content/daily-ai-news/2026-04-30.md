## 1. 旗舰模型发布

### 🔥 Mistral Medium 3.5 与 Vibe 远程编码 Agent 同步上线，合并 Magistral 与 Devstral 2
Mistral AI 于 4 月 30 日发布 **Mistral Medium 3.5**，以单一权重取代此前 Magistral（推理）与 Devstral 2（编码）两条产品线。规格为 **128B dense 参数、256K** 上下文，可单 4 卡部署，基准侧 SWE-Bench Verified **77.6%**、τ³-Telecom 91.4%。API 定价 **$1.50 / $7.50 per MTok**，权重在 Hugging Face 以修改版 MIT 协议开放下载。同日上线 Vibe 远程编码 Agent：云端 sandbox 异步运行，可从 CLI 或 Le Chat 启动，原生对接 GitHub、Linear、Sentry、Slack。Mistral 不到 30 天前借款 $8.3 亿建设 13,800 GPU 数据中心，Medium 3.5 为该集群首个旗舰输出。（来源：[Mistral AI Vibe Agents](https://mistral.ai/news/vibe-remote-agents-mistral-medium-3-5) · [Hugging Face Mistral Medium 3.5](https://huggingface.co/mistralai/Mistral-Medium-3.5-128B) · [Open Data Science](https://opendatascience.com/mistral-launches-medium-3-5-and-cloud-based-remote-agents-for-async-coding/)）

### 🔥 xAI Grok 4.3 API 全量发布：1M 上下文、原生视频输入、文件输出
xAI 于 4 月 30 日完成 **Grok 4.3** API 全量推送，开发者控制台提示从 grok-4.20 迁移。上下文窗口提升至 **1M token**，多模态侧首次支持原生视频输入（最长 5 分钟 1080p MP4、MOV、WebM）；输出侧可直接生成 PDF、PPTX、XLSX 文件。API 定价 **$1.25 / $2.50 per MTok**，输入价较 Grok 4 下调约 **40%**。Artificial Analysis Intelligence Index 评测得 **53 分**，低于 GPT-5.5 的 60 分与 Claude Opus 4.7 的 57 分，但被评价「重写了 agentic 应用的成本曲线」。xAI 同日上线 voice agent API，含声音克隆与 console 声音目录管理。（来源：[VentureBeat Grok 4.3](https://venturebeat.com/technology/xai-launches-grok-4-3-at-an-aggressively-low-price-and-a-new-fast-powerful-voice-cloning-suite) · [Artificial Analysis Grok 4.3](https://artificialanalysis.ai/articles/xai-launches-grok-4-3-with-improved-agentic-performance-and-lower-pricing) · [OpenRouter Grok 4.3](https://openrouter.ai/x-ai/grok-4.3)）

### Anthropic 1M 上下文 beta 4 月 30 日退役，统一迁移至 Sonnet 4.6 与 Opus 4.6
Anthropic 于 4 月 30 日正式退役 **Claude Sonnet 4.5 与 Sonnet 4** 的 1M token 上下文 beta，`context-1m-2025-08-07` 头部对该两款模型自此失效，超过 200K token 的请求返回 **422 错误**。Claude Sonnet 4.6、Opus 4.6 与 4 月 17 日发布的 Opus 4.7 均以标准价格原生支持 1M 上下文，无需 beta 头部、无长上下文价格溢价。Opus 4.7 定价 $5 / $25 per MTok，SWE-bench Verified 报 **87.6%**。退役推动前沿厂商在「1M 上下文标配化」上分野收敛，对照 Grok 4.3、Gemini 3.1 与 Mistral Medium 3.5（256K）。

## 4. AI Agent 基础设施

### 🔥 Cloudflare 发布企业 MCP 参考架构，Code Mode 削减 token 用量约 99.9%
Cloudflare 于 4 月 30 日在 Agents Week 系列发布**企业级 MCP 参考架构**与 Code Mode 通用版。架构层面采用集中部署、单一团队治理，借助 Cloudflare Access、AI Gateway 与 MCP server portals 形成统一发现入口；并通过 Cloudflare Gateway 识别 JSON-RPC 流量、阻断未授权的「影子 MCP」。Code Mode 把 MCP server schema 自动转译为带文档注释的 TypeScript API：在 2,500+ API 端点场景下，token 占用从约 **117 万**降至约 **1,000**，下降约 **99.9%**。同日还发布 13 个新 MCP 服务器，并公开 Cloudflare 内部 AI 工程栈作为参考实施。（来源：[Cloudflare Enterprise MCP](https://blog.cloudflare.com/enterprise-mcp/) · [Cloudflare Code Mode](https://blog.cloudflare.com/code-mode/) · [InfoQ Cloudflare Code Mode](https://www.infoq.com/news/2026/04/cloudflare-code-mode-mcp-server/)）

### 🔥 Okta for AI Agents 4 月 30 日 GA，定位为 Agent 身份层
Okta 在 Showcase 2026 大会按 3 月 21 日预告时点上线 **Okta for AI Agents** 通用版，回答企业三个问题：Agent 在哪、能连接什么、能做什么。Universal Directory 把 AI Agent 视为一等非人身份并记录全生命周期；Agent Gateway 作为集中控制平面，把 Okta MCP 注册表汇聚为虚拟 MCP server 暴露给 Agent，所有交互落审计。平台默认对接 MCP、A2A、Anthropic Managed Agents、Microsoft Copilot Studio、Google Gemini Enterprise 等框架。Okta 同日公布研究：**88%** 受访企业曾报告 AI Agent 相关安全事件，但仅 **22%** 将 Agent 视作独立身份对象。（来源：[Okta Showcase 2026](https://www.okta.com/newsroom/press-releases/showcase-2026/) · [Okta 投资者公告](https://investor.okta.com/news-and-events/news-releases/news-details/2026/Okta-Announces-New-Blueprint-for-the-Secure-Agentic-Enterprise/default.aspx) · [Stock Titan OKTA 4/30 GA](https://www.stocktitan.net/news/OKTA/okta-announces-new-blueprint-for-the-secure-agentic-zimmq0wegfj3.html)）

## 9. 政策与监管

### 🔥 EU AI Act Digital Omnibus 三方会议破裂，8 月 2 日合规期再度逼近
欧洲议会、理事会与委员会于 4 月 28 日深夜起进行约 12 小时的 Digital Omnibus 三方会议，4 月 29 日凌晨宣布无果，4 月 30 日多家律所与 IAPP 集中发布企业影响分析。破裂焦点并非延期方案，而是 Annex I 嵌入式 AI（医疗设备、玩具、连网车、工业机械）是否仅受部门法规管辖、绕开 AI Act 附加义务的合规架构争议。原拟方案曾将 Annex III 延至 2027 年 12 月、Annex I 延至 2028 年 8 月，本次未通过意味着 **2026 年 8 月 2 日** Annex III 合规期按原时间表生效，覆盖信用评分、招聘、司法、边境等高风险系统，距 4 月 30 日仅 **95 天**。下一轮预计 5 月中旬召开，塞浦路斯主席国任期至 6 月 30 日为最后窗口。（来源：[IAPP AI Act Omnibus](https://iapp.org/news/a/ai-act-omnibus-what-just-happened-and-what-comes-next) · [Bird & Bird Digital Omnibus Stalls](https://www.twobirds.com/en/insights/2026/germany/digital-omnibus-on-ai-trilogue-stalls-ahead-of-the-ai-act-deadline) · [Result Sense EU AI Act Stalls](https://www.resultsense.com/news/2026-04-30-eu-ai-act-omnibus-trilogue-stalls)）
