## 3. AI 产品更新

### 🔥 Meta 在 WhatsApp 上线 Incognito Chat with Meta AI，基于 TEE 的"自家也看不到"私聊

Meta 5/13 在 WhatsApp blog 与 about.fb.com newsroom 同步发布 **Incognito Chat with Meta AI**：与 Meta AI 的对话会话结束默认消失，不进入广告定向与模型训练。底层依赖 WhatsApp 自研的 Private Processing 栈，推理跑在 TEE 里——Meta 自家工程师与基础设施无法读取处理中的 prompt。

差异点在架构层：其他厂商的 incognito 通常只是 retention 策略变更（仍可读 prompt），Meta 这次是用 enclave 在技术层阻断 provider 访问。首发限文本、暂不支持图像；分阶段在 WhatsApp 与 Meta AI app 推出。

（来源：[WhatsApp Blog](https://blog.whatsapp.com/introducing-incognito-chat-with-meta-ai-a-completely-private-way-to-chat-with-ai) · [About Meta](https://about.fb.com/news/2026/05/incognito-chat-whatsapp-meta-ai/) · [TechCrunch](https://techcrunch.com/2026/05/13/whatsapp-adds-an-incognito-mode-in-meta-ai-chats/)）

### Cursor 3.4 发布：云 Agent 的多 repo 开发环境 + Dockerfile 构建缓存

Cursor 5/13 推出 **3.4**：核心是为 cloud agent 和 automations 引入可重用的多 repo 开发环境，每个环境可包含克隆好的多个仓库、安装好的依赖、内部工具链凭证与构建系统访问。

实现细节：环境用 Dockerfile 描述；新增 build secrets，仅在构建步骤生效、不会传入运行中的 agent；升级 layer caching，命中缓存的重建快 ~70%。每个环境有独立的版本历史与回滚；Cursor 自身在配置过程会发问、标出缺失凭证、验证设置。配合此前 3.2/3.3 的 agent context 改造，目标是把团队跑大量并行 agent 的成本降下来。

（来源：[Cursor Changelog 05-13-26](https://cursor.com/changelog/05-13-26) · [Cursor Blog: Cloud Agent Dev Environments](https://cursor.com/blog/cloud-agent-development-environments)）

### Codex 进入 ChatGPT 移动端预览

OpenAI 5/13 起将 **Codex preview** 推入 ChatGPT iOS 与 Android app，覆盖 Free 与 Go 在内的全部档位（按地区可用性）。手机上可以发起或续写 Codex 线程、批准动作、查看运行结果与 diff/截图/测试结果；Codex 后台跑在已连接的 Mac 上。Mac 侧 9to5Mac 5/14 跟进的功能细节是手机端的指挥面板与 Mac 端实际算力的解耦。

（来源：[Codex Changelog](https://developers.openai.com/codex/changelog) · [9to5Mac](https://9to5mac.com/2026/05/14/openai-brings-codex-control-to-chatgpt-for-iphone-and-android/)）
