## 2. AI Agent 与编码工具

### OpenAI 收购云平台 Ona（原 Gitpod），强化 Codex 长任务

OpenAI 借 Ona 让 Codex agent 在关机后续跑，剑指周活 **500 万**用户的留存与扩张。**OpenAI** 6/11 宣布收购德国云初创 **Ona**（原 **Gitpod GmbH**），将其安全云沙箱与 agent 编排技术并入 **Codex**；交易金额未披露、尚待监管批准，Ona CEO Johannes Landgraf 及团队加入 Codex 组。Codex 周活已破 500 万、同比 **+400%**，Ona 沙箱可在开发者关闭工作站后维持 agent 任务不中断。此举把 OpenAI 推向 Anthropic 主攻的企业编码市场正面竞争。下一关键节点：交易交割（受惯例成交条件与监管审批约束）后，Ona 长任务能力并入 Codex 的落地时点。

（来源：[OpenAI](https://openai.com/index/openai-to-acquire-ona/) · [Bloomberg](https://www.bloomberg.com/news/articles/2026-06-11/openai-to-acquire-cloud-platform-ona-to-support-ai-agents) · [CNBC](https://www.cnbc.com/2026/06/11/open-ai-ona-acquisition-codex.html)）

### xAI 上线 Grok Build 插件市场，首发 6 款合作插件

本周编码 agent 三方过招：xAI 开插件市场、OpenAI 购 Ona、Claude Code 加嵌套子代理。**xAI** 6/11 为其终端编码工具 **Grok Build** 上线内置**插件市场**，首发 **MongoDB**、Vercel、Sentry、Chrome DevTools、Cloudflare、Superpowers 共 **6** 款合作插件，开发者可在终端内 `/marketplace` 浏览安装、亦可提交 PR 发布自有插件。这把编码 agent 竞争从模型能力延伸到插件生态。下一关键节点：开放目录（xai-org/plugin-marketplace）第三方插件数能否快速增长，是其生态吸引力的首个检验。

（来源：[xAI](https://x.ai/news/grok-plugin-marketplace) · [MarkTechPost](https://www.marktechpost.com/2026/06/11/xai-ships-grok-build-plugin-marketplace-with-mongodb-vercel-sentry-chrome-devtools-cloudflare-and-superpowers-plugins-at-launch/)）
