### 🧠 大语言模型（LLM）/ 推理能力

**OpenDeepThink: Parallel Reasoning via Bradley–Terry Aggregation**
📄 https://arxiv.org/abs/2605.15177 ｜ 💻 暂未开源 ｜ 🤗 HF trending ｜ 提交：2026-05-14（v2 修订 05-17）

**问题**
测试时计算（test-time compute）扩展「广度」的标准做法是并行采样多个候选答案，但这引入了一个**选择瓶颈**：在没有 ground-truth verifier 的开放任务上，如何挑出最优候选？此前依赖的 pointwise LLM 打分（让模型对单个答案直接评分）既有噪声又有偏差，分数不可比、易被表述风格带偏，导致并行采样的算力没有转化为有效收益。

**方法**
- **核心机制**：把候选答案视为一个「种群」，用 population-based 的进化式搜索替代一次性采样。每一代中，LLM 不做绝对评分，而是对随机抽取的候选**配对**进行两两比较；
- **Bradley–Terry 聚合**：将大量成对比较结果用 Bradley–Terry 模型聚合成一个全局排名——这是关键设计决策，配对判断比绝对评分更稳健、可消去打分尺度漂移；
- **进化算子**：保留排名靠前的候选，对前 3/4 的候选利用比较过程中自然产生的**自然语言批评（critique）**进行变异（mutation），淘汰末位 1/4。批评既是排序信号也是改写指导，二者复用同一次 LLM 调用。
- **与已有方法的区别**：相比 best-of-N + pointwise judge，OpenDeepThink 把「选择」和「改进」统一进配对比较循环，避免了绝对打分的偏差；相比单纯多数投票，它保留了语义层面的迭代精炼。

**效果**
- 在 8 轮顺序 LLM 调用（约 27 分钟 wall-clock）内，将 Gemini 3.1 Pro 的有效 Codeforces Elo 提升 **+405 分**；
- 表明在缺乏可验证奖励的竞赛级编程任务上，配对聚合式的测试时计算扩展比 pointwise 选择具有实质优势。

**How Off-Policy Can GRPO Be? μ-GRPO for Efficient LLM Reinforcement Learning**
📄 https://arxiv.org/abs/2605.17570 ｜ 💻 暂未开源 ｜ 机构：见论文 ｜ 提交：2026-05-17

**问题**
GRPO 是当前 RLVR（可验证奖励强化学习）的主力算法，但通常运行在「低 staleness、近 on-policy」的区间内：rollout 生成与策略优化必须频繁切换，训练器在两者之间反复等待，带来巨大的系统开销（generation-optimization 切换成本）。问题在于——这种近 on-policy 约束究竟是算法必需，还是工程惯性？

**方法**
- **核心设计**：μ-GRPO 把训练重组为少数（如 4 个）**大型顺序化的 generation-optimization 阶段**，每个阶段批量生成、批量优化。这会主动引入**高 rollout staleness**（用陈旧数据训练），但大幅削减切换开销；
- **稳定化技术一——relaxed clipping**：放宽 PPO 式裁剪边界，保留来自陈旧 rollout 的有用梯度，而非将其裁掉；
- **稳定化技术二——negative-advantage veto**：对负优势（negative-advantage）响应中触发点之后的后缀更新进行否决，移除会破坏训练稳定性的更新项；
- **与 GRPO 的本质区别**：标准 GRPO 假设必须近 on-policy；μ-GRPO 论证 GRPO 类算法能容忍**远高于此前假设**的 staleness，从而把训练系统效率与算法正确性解耦。

**效果**
- 论文核心结论为定性突破：证明 GRPO 风格算法可在显著更高的 rollout staleness 下保持稳定学习，意味着可大幅减少 rollout-optimization 切换、提升训练吞吐。具体加速比与下游 benchmark 数字建议以论文正文 / 表格为准（本周仅获取到 v1 摘要级信息，未逐项核到表内数值）。

> 同方向另值得关注：`2605.21468`（Minimal RLVR Training，提出 RLVR 权重轨迹极度低秩、可用 rank-1 线性外推未来 checkpoint）、`2605.16787`（RLVR 中的「不可学习」现象）。

### 👁️ 多模态（图像 / 视频 / 音频）

**Lance: Unified Multimodal Modeling by Multi-Task Synergy**
📄 https://arxiv.org/abs/2605.18678 ｜ 💻 暂未开源 ｜ 提交：2026-05-18

**问题**
统一多模态模型（理解 + 生成 + 编辑）此前往往为每个任务设定固定的输入-输出映射、或训练分离的子模型，难以建模跨模态、跨任务之间的相关性；同时「原生统一」模型通常体量庞大、部署成本高。

**方法**
- **轻量原生统一**：Lance 是一个轻量级的原生统一模型，在同一架构内同时支持图像与视频的多模态理解、生成与编辑；
- **协同多任务训练**：在共享的交错多模态序列（interleaved multimodal sequences）上做协同训练，让不同任务互为正则、共享表征；
- **双流 MoE 架构**：采用 dual-stream mixture-of-experts，对理解与生成两类信号分流处理，再在共享序列上协同——这是其在保持轻量的同时覆盖多任务的关键设计。

**效果**
- 论文主张通过多任务协同训练，在统一架构下取得理解、生成、编辑的均衡表现；具体 benchmark 数值建议以论文正文为准。

> 同方向参考：`2605.02641`（Mamoda2.5，AR-Diffusion 统一框架，DiT-MoE 主干 128 专家 Top-8 路由、25B 参数，在 VBench 2.0 上取得领先生成表现）。

### 🤖 AI Agent / 工具使用

**AgentTrust: Runtime Safety Evaluation and Interception for AI Agent Tool Use**
📄 https://arxiv.org/abs/2605.04785 ｜ 💻 暂未开源 ｜ 提交：2026-05-05

**问题**
现代 AI Agent 通过工具调用执行真实世界副作用——文件操作、shell 命令、HTTP 请求、数据库查询。问题在于这些副作用在执行**之后**才被发现，缺乏在执行**之前**进行风险判定与拦截的运行时机制。

**方法**
- **运行时安全层**：AgentTrust 在 Agent 的每次工具调用真正执行**之前**进行拦截；
- **结构化判定**：对每次调用返回 allow / warn / block / review 四类结构化裁决，把安全策略从「事后审计」前移为「执行前门控」；
- **与已有方法区别**：不同于对模型输出做静态过滤或事后日志分析，AgentTrust 作用在「工具调用边界」这一可观测、可干预的关键位点。

**效果**
- 论文给出在该拦截框架下对高风险工具调用的判定与阻断能力评估；具体指标建议以论文正文为准。该工作与本周另两篇 Agent 论文（`2605.06890` 用机制可解释性做工具调用内部可观测、`2605.18747` 综述「Code as Agent Harness」）共同指向同一趋势：Agent 工程的重心正从「能不能调工具」转向「调用是否可控、可验证、可观测」。

### 🦾 具身智能 / 机器人

**World Action Models: The Next Frontier in Embodied AI（综述）**
📄 https://arxiv.org/abs/2605.12090 ｜ 💻 暂未开源 ｜ 机构：复旦大学（Yu-Gang Jiang、Xipeng Qiu 等）｜ 提交：2026-05-12

**问题**
Vision-Language-Action（VLA）模型在具身策略学习上有很强的语义泛化，但它们学到的是**反应式的「观测→动作」映射**，没有显式建模「物理世界在干预下如何演化」。引入 world model（环境动力学预测模型）的工作日益增多，但散落在不同架构、学习目标与应用场景中，缺乏统一概念框架。

**方法（综述贡献）**
- **正式定义 World Action Models（WAMs）**：统一「预测式状态建模」与「动作生成」的具身基础模型，目标是对「未来状态 + 动作」的**联合分布**建模，而非仅建模动作；
- **结构化分类法**：将现有方法划分为 Cascaded WAMs 与 Joint WAMs，再按生成模态、条件机制、动作解码策略进一步细分；
- **数据生态梳理**：系统分析支撑 WAMs 的数据来源——机器人遥操作、便携式人类示范、仿真、互联网规模第一人称视频；
- **评测协议**：归纳出围绕视觉保真度、物理常识、动作合理性三个维度的新兴评测范式。

**效果（综述价值）**
- 提供了 WAMs 领域的首份系统性梳理，厘清了主要架构范式及其取舍，并指出开放挑战；对正在从 VLA 向「world model + action」范式迁移的具身研究团队有路线图意义。

### 🔬 AI for Science

**SPADE: Faster Drug Discovery by Learning from Sparse Data**
📄 https://arxiv.org/abs/2605.05370 ｜ 💻 暂未开源 ｜ 作者：Rahul Nandakumar、Ben Fauber、Deepayan Chakrabarti ｜ 提交：2026-05-06

**问题**
药物发现中，候选配体（ligand）通过早期筛选门槛的比例不足 5%，且方法必须对**没有任何先验数据的全新蛋白靶点**有效。深度学习与贝叶斯优化方法在这种「稀疏数据 + 冷启动」场景下样本效率与速度都不理想。

**方法**
- **从稀疏数据学习的配体选择**：SPADE 针对「测试预算极其有限」的设定设计配体选择策略，目标是用尽可能少的湿实验/打分测试锁定高质量配体；
- **冷启动友好**：方法设计上不依赖目标蛋白的历史数据，适配全新蛋白靶点；
- **与 DL / 贝叶斯优化的区别**：在相同测试预算下提供更高的样本效率，并在打分阶段大幅降低计算成本。

**效果**
- 平均仅需 **40 次测试**即可找到 10 个高质量配体；
- 在一对一比较中，SPADE 在更多蛋白上优于深度学习与贝叶斯优化方法，样本效率中位数提升 **7%–32%**；
- 在候选药物打分速度上比最接近的竞争方法**快约 10 倍**。

**ToolMol: Evolutionary Agentic Framework for Multi-objective Drug Discovery**
📄 https://arxiv.org/abs/2605.12784 ｜ 💻 暂未开源 ｜ 机构：UC San Diego（Rose Yu 等）｜ 提交：2026-05-12（v2 05-14）

**问题**
基于 LLM 的分子生成在 de novo 药物设计上很有前景，但现有方法生成的配体候选**无效率高、质量低**——根源是当前模型对分子字符串（SMILES 等）的**句法处理能力有限**，容易产出语法非法或不可合成的结构。

**方法**
- **进化式 Agentic 框架**：ToolMol 将多目标遗传算法与一个 agentic LLM 算子结合，由 LLM 算子迭代更新配体种群；
- **RDKit 工具箱**：构建一套基于 RDKit 的函数工具箱，让 agentic 算子通过**工具调用**而非直接生成字符串来做精确的配体修改——这是规避 SMILES 句法缺陷的关键设计；
- **与纯 LLM 生成的本质区别**：把「分子编辑」从「生成合法字符串」问题转化为「调用确定性化学函数」问题，LLM 只负责规划修改、由工具保证化学合法性。论文通过分析 CoT 推理轨迹发现，工具调用使模型更忠实地执行其规划的修改，更有效地利用 LLM 中的化学先验。

**效果**
- 在多目标性质优化任务上达到 SOTA，于 3 个蛋白靶点上发现的类药、可合成配体，预测结合亲和力比现有方法**强 10% 以上**；
- 在 gold-standard 的 Absolute Binding Free Energy（绝对结合自由能）评分上同样取得 SOTA，较现有方法提升**超过 35%**。

### 🛡️ AI 安全 / 对齐 / 可解释性

**Understanding Annotator Safety Policy with Interpretability**
📄 https://arxiv.org/abs/2605.05329 ｜ 💻 暂未开源 ｜ 提交：2026-05

**问题**
安全数据的标注质量直接决定对齐效果，但不同标注者「内心的安全策略」是不可见的——同一条安全指令会被不同人不同解读，这种策略歧义与价值多元很难量化，也无法在不增加额外标注成本的前提下做对比。

**方法**
- **Annotator Policy Models（APMs）**：仅从标注者的标注行为本身学习一个可解释模型，用于刻画该标注者隐含的安全策略，使其推理「可见、可比较」；
- **无需额外标注**：APMs 不要求标注者额外解释，直接从已有标签反推策略；
- **两类应用**：(1) 暴露**策略歧义**——揭示不同标注者如何对同一安全指令做出不同解读；(2) 暴露**价值多元**——发现不同人群在安全优先级上的系统性差异。

**效果**
- APMs 能以 **>80% 准确率**建模标注者的安全策略，可忠实预测其对反事实编辑（counterfactual edits）的响应，并在受控设置下复现已知的策略差异。

### ⚡ 高效推理 / 量化 / 压缩

**FibQuant: Universal Vector Quantization for Random-Access KV-Cache Compression**
📄 https://arxiv.org/abs/2605.11478 ｜ 💻 暂未开源 ｜ 作者：Namyoon Lee、Yongjune Kim ｜ 提交：2026-05-12

**问题**
长上下文推理本质上是一个**显存带宽**问题：KV cache 随上下文长度、batch、层数、头数增长，且在每个解码步都要被读取。基于旋转的标量编解码器（rotation-based scalar codec）——存范数、做共享随机旋转、逐坐标标量量化——虽然通用且支持随机访问，但它**丢弃了归一化步骤产生的几何结构**：在 Haar 旋转后，连续 $k$ 个坐标并不是一个乘积源（product source），而是单位球上的 spherical-Beta 源。标量量化无视这一结构，在低比特下迅速失效。

**方法**
- **核心思想**：FibQuant 保留「normalize–rotate–store」这一通用且随机访问友好的接口，但把逐坐标的标量码表替换为一个**共享的径向-角向码本（radial–angular codebook）**，使之匹配 spherical-Beta 这一规范源；
- **码本构造**：径向用 Beta 分位数半径（Beta-quantile radii），角向用 Fibonacci / Roberts–Kronecker 拟均匀方向，再用 multi-restart Lloyd–Max 精修；
- **理论保证**：证明该向量码在相同码率下严格优于其标量乘积特化，高码率增益可分解为「单元形状因子（cell-shaping）」与「密度匹配因子（density-matching）」两项；
- **与已有方法的本质区别**：同样维持通用 + 随机访问接口（这是部署侧硬约束），但通过向量量化夺回了标量编解码器丢弃的球面几何；并提供连续码率轴，支持分数比特乃至亚 1-bit 工作点，**无需校准、无需变长地址**。

**效果**
- 在 GPT-2 small 的 KV cache 上，描绘出一条「显存-保真」前沿：从 5× 压缩（attention 余弦相似度 0.99）到 34× 压缩（相似度 0.95）；
- 端到端在 TinyLlama-1.1B 上：4× 压缩时 perplexity 与 fp16 相差仅 **0.10**；
- 在 $b=2$（8× 压缩，标量随机访问量化开始失效的区间）时，perplexity 比标量方法 TurboQuant **低 3.6 倍**。

> 同方向参考：`2605.20868`（Runtime-Certified Bounded-Error Quantized Attention，分层 KV cache——GPU 内存存 INT8 keys + INT4 values，系统内存保留 FP16 原值作确定性回退）。
